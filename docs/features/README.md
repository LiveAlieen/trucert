# 功能说明

## 1. 密钥管理

### 1.1 密钥对生成

系统支持生成两种类型的密钥对：

- **RSA密钥对**：支持多种密钥长度（2048、3072、4096位）
- **ECC密钥对**：支持多种椭圆曲线（secp256r1、secp384r1、secp521r1）

**生成流程**：
1. 用户选择密钥类型（RSA或ECC）
2. 用户选择密钥参数（密钥长度或椭圆曲线）
3. 系统生成密钥对
4. 系统将密钥对存储到指定位置

### 1.2 密钥存储

密钥存储采用以下策略：

- **私钥**：以PEM格式存储在安全目录中
- **公钥**：以PEM格式存储，可公开访问
- **密钥元数据**：存储密钥的相关信息，如生成时间、算法类型等

**存储结构**：
```
configs/key/{key_id}/
├── {key_id}_private.pem    # 私钥文件
├── {key_id}_public.pem     # 公钥文件
└── {key_id}_metadata.json  # 密钥元数据
```

### 1.3 密钥加载

系统支持从存储中加载已有的密钥对，用于后续的签名、验证或证书生成操作。

### 1.4 密钥管理

- **列出密钥**：显示所有已存储的密钥信息
- **删除密钥**：从存储中删除指定的密钥对
- **根密钥管理**：专门管理用于签发证书的根密钥对

## 2. 证书管理

### 2.1 自签名证书生成

自签名证书是由密钥所有者自己签发的证书，主要用于测试环境或作为根证书。

**生成流程**：
1. 用户选择或生成一个密钥对
2. 用户提供证书主题信息（如通用名称、组织等）
3. 系统使用私钥签发证书
4. 系统将证书存储到指定位置

### 2.2 二级证书生成

二级证书是由其他证书（如根证书）签发的证书，形成证书链结构。

**生成流程**：
1. 用户选择或生成一个密钥对作为证书的公钥
2. 用户选择一个签发证书（如根证书）
3. 用户提供证书主题信息
4. 系统使用签发证书的私钥签发生成二级证书
5. 系统将证书存储到指定位置

### 2.3 证书存储

证书以JSON格式存储，包含以下信息：

- **证书主题**：证书持有者的信息
- **证书颁发者**：签发证书的实体信息
- **公钥**：证书对应的公钥
- **有效期**：证书的有效时间段
- **签名算法**：用于签发证书的算法
- **签名值**：证书的数字签名

**存储结构**：
```
configs/trust/
├── self_signed_{timestamp}.json   # 自签名证书
└── secondary_{timestamp}.json     # 二级证书
```

### 2.4 证书加载

系统支持从存储中加载已有的证书，用于后续的验证或签发操作。

### 2.5 证书管理

- **列出证书**：显示所有已存储的证书信息
- **删除证书**：从存储中删除指定的证书
- **导入证书**：从外部文件导入证书到系统中

## 3. 文件签名

### 3.1 文件签名功能

系统支持使用私钥对文件进行数字签名，确保文件的完整性和真实性。

**签名流程**：
1. 用户选择要签名的文件
2. 用户选择用于签名的私钥
3. 系统计算文件的哈希值
4. 系统使用私钥对哈希值进行签名
5. 系统生成签名文件

### 3.2 签名文件格式

签名文件以JSON格式存储，包含以下信息：

- **文件路径**：被签名文件的路径
- **文件哈希**：文件的哈希值
- **签名值**：使用私钥生成的签名
- **签名时间**：签名操作的时间戳
- **签名算法**：使用的签名算法

**存储结构**：
```
configs/key/{key_id}/
└── {key_id}_signature.txt  # 签名文件
```

## 4. 文件验证

### 4.1 签名验证功能

系统支持使用公钥或证书验证文件签名的有效性，确保文件未被篡改。

**验证流程**：
1. 用户选择要验证的文件和对应的签名文件
2. 用户选择用于验证的公钥或证书
3. 系统计算文件的哈希值
4. 系统使用公钥或证书中的公钥验证签名
5. 系统返回验证结果

### 4.2 验证结果

验证结果包括以下几种情况：

- **验证成功**：文件未被篡改，签名有效
- **验证失败**：文件已被篡改，签名无效
- **证书过期**：用于验证的证书已过期
- **证书无效**：用于验证的证书本身无效

## 5. 配置管理

### 5.1 配置存储

系统支持将配置信息以JSON格式存储，便于后续加载和使用。

**存储结构**：
```
configs/
├── algorithms.json      # 算法配置
└── cert_versions.json   # 证书版本配置
```

### 5.2 配置类型

系统支持以下类型的配置：

- **算法配置**：定义支持的哈希算法、RSA密钥大小和ECC曲线
- **证书版本配置**：定义不同版本证书的字段结构
- **系统配置**：定义系统的全局设置

### 5.3 配置管理

- **加载配置**：从存储中加载配置信息
- **保存配置**：将配置信息保存到存储中
- **更新配置**：更新现有配置的部分内容
- **获取配置**：获取配置信息，如果不存在则返回默认值

## 6. 存储管理

### 6.1 存储接口

系统提供统一的存储接口，支持以下操作：

- **保存数据**：将数据保存到文件（支持JSON和二进制格式）
- **加载数据**：从文件加载数据
- **删除文件**：删除指定的文件
- **列出文件**：列出目录中的文件
- **获取文件信息**：获取文件的详细信息

### 6.2 存储目录

系统管理以下存储目录：

- **基础配置目录**：存储系统配置文件
- **密钥存储目录**：存储密钥对和相关信息
- **证书信任目录**：存储证书文件
- **根密钥目录**：存储根密钥对

### 6.3 存储安全性

- **目录权限**：确保存储目录具有适当的权限设置
- **数据完整性**：验证存储数据的完整性
- **错误处理**：妥善处理存储操作中的错误情况

## 7. 日志管理

系统提供全面的日志记录功能，包括：

- **操作日志**：记录系统的主要操作
- **错误日志**：记录系统运行中的错误
- **调试日志**：记录详细的调试信息

**日志级别**：
- DEBUG：详细的调试信息
- INFO：一般的信息性消息
- WARNING：警告消息
- ERROR：错误消息
- CRITICAL：严重错误消息

## 8. 错误处理

系统具有完善的错误处理机制，包括：

- **参数验证**：验证输入参数的有效性
- **异常捕获**：捕获并处理运行时异常
- **错误消息**：提供清晰、详细的错误消息
- **边界条件处理**：处理各种边界情况

**常见错误类型**：
- **文件操作错误**：文件不存在、权限不足等
- **加密操作错误**：密钥无效、算法不支持等
- **参数错误**：参数格式不正确、值无效等
- **配置错误**：配置文件不存在、格式错误等

## 9. 功能集成

系统的各个功能模块可以集成使用，形成完整的证书管理流程：

1. **完整证书流程**：生成密钥对 → 生成自签名证书 → 生成二级证书 → 使用证书进行文件签名和验证
2. **安全通信支持**：为TLS/SSL等安全通信协议提供证书支持
3. **身份验证流程**：使用证书进行身份验证，确保通信双方的身份

## 10. 性能优化

系统在设计和实现中考虑了性能优化，包括：

- **内存使用**：合理管理内存使用，避免内存泄漏
- **I/O操作**：优化文件I/O操作，减少磁盘访问
- **计算效率**：选择高效的算法和实现方式
- **并发处理**：支持并发操作，提高系统吞吐量

系统设计具有良好的可扩展性，包括：

- **模块化结构**：采用模块化设计，便于添加新功能
- **插件机制**：支持通过插件扩展系统功能
- **配置驱动**：通过配置文件控制系统行为
